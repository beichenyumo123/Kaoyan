# 功能拓展总结

本文档记录在考研论坛基础功能之上，围绕**安全性、可用性、可扩展性**三方面进行的功能拓展。

---

## 一、Redis 接入（缓存与限流基础）

**目标**：为后续接口限流和业务缓存提供 Redis 基础设施。

### 变更清单

| 文件 | 动作 | 说明 |
| --- | --- | --- |
| `pom.xml` | 修改 | 新增 `spring-boot-starter-data-redis` + `commons-pool2` |
| `application.properties` | 修改 | 新增 Redis 连接 + Lettuce 连接池配置 |
| `config/RedisConfig.java` | **新建** | RedisTemplate 和 StringRedisTemplate Bean 配置 |

### 技术选型

- **客户端**：Lettuce（Spring Boot 3 默认，Netty 异步、线程安全）
- **序列化方案**：
  - Key → `StringRedisSerializer`（避免 Redis 中的 `\xac\xed\x00\x05` 乱码）
  - Value → `GenericJackson2JsonRedisSerializer`（JSON 格式，注册 `JavaTimeModule` 支持 `LocalDateTime`）
- **未启用 `defaultTyping`**：不注入 `@class` 字段，数据更干净
- **ObjectMapper 独立构造**：不污染前端 Jackson 配置（Long → String 等）

### 连接池配置

```properties
spring.data.redis.host=localhost
spring.data.redis.port=6379
spring.data.redis.lettuce.pool.max-active=16
spring.data.redis.lettuce.pool.max-idle=8
spring.data.redis.lettuce.pool.min-idle=2
```

## 二、接口限流（防刷）

**目标**：基于 Redis 实现接口级别的滑动窗口限流，防止恶意刷接口。

### 变更清单

| 文件 | 动作 | 说明 |
| --- | --- | --- |
| `pom.xml` | 修改 | 新增 `spring-boot-starter-aop` 依赖 |
| `common/result/ResultCode.java` | 修改 | 新增 `RATE_LIMITED(429, "请求过于频繁，请稍后再试")` |
| `common/annotation/RateLimit.java` | **新建** | `@RateLimit` 注解（参数：time、maxCount、message） |
| `common/aspect/RateLimitAspect.java` | **新建** | AOP 切面，Lua 脚本 + Redis ZSET 滑动窗口 |

### 核心设计

```
请求 → @Around("@annotation(rateLimit)")
        → 构建 Key: rate_limit:{类名}:{方法名}:{userId 或 IP}
        → Lua 原子脚本: ZREMRANGEBYSCORE → ZCARD → ZADD / 拒绝
        → 通过: proceed()
        → 拒绝: throw BusinessException(429)
        → GlobalExceptionHandler 统一返回 {"code":429,...}
```

### 技术细节

- **算法**：ZSET 滑动窗口（非固定窗口），消除边界突增问题
- **原子性**：Lua 脚本在 Redis 服务端原子执行，避免并发竞态
- **用户标识**：已登录按 `userId` 限流，未登录按 IP 限流（X-Forwarded-For / X-Real-IP / RemoteAddr）
- **Key 隔离**：不同 Controller、不同方法、不同用户/IP 的限流互相独立
- **Key 自动过期**：窗口时间结束后 Redis Key 自动清除，不积累垃圾数据

### 使用示例

```java
@RateLimit(time = 60, maxCount = 3, message = "发帖太频繁，请稍后再试")
@PostMapping
public Result<Long> createPost(@Validated @RequestBody PostDTO postDTO) { ... }
```

### 测试验证

```bash
curl localhost:8081/api/boards  # 1 → 200
curl localhost:8081/api/boards  # 2 → 200
curl localhost:8081/api/boards  # 3 → 429 "测试限流"
```

## 三、XSS 安全防护

**目标**：对前端传入的所有 JSON String 字段自动清洗 XSS 攻击载荷，同时为富文本保留安全 HTML 标签。

### 变更清单

| 文件 | 动作 | 说明 |
| --- | --- | --- |
| `pom.xml` | 修改 | 新增 `jsoup 1.18.1` 依赖 |
| `common/annotation/SkipXssClean.java` | **新建** | 标注字段跳过 XSS 清洗（如 password） |
| `common/handler/XssStringDeserializer.java` | **新建** | Jackson 全局 String 反序列化器 |
| `config/JacksonConfig.java` | 修改 | 注册 XssCleanModule Bean |
| `module/auth/entity/RegisterDTO.java` | 修改 | `password` 字段加 `@SkipXssClean` |
| `module/auth/entity/LoginDTO.java` | 修改 | `password`、`account` 字段加 `@SkipXssClean` |

### 核心设计

```
请求 JSON → Jackson 反序列化
           → 遇到 String 类型 → XssStringDeserializer
           → 检测 @SkipXssClean 注解
              ├─ 有注解 → 跳过清洗（适用于 password）
              └─ 无注解 → Jsoup.clean(value, Safelist.relaxed())
           → Controller 拿到已清洗的 DTO
```

### 技术选型对比

| 方案 | 优点 | 缺点 |
| --- | --- | --- |
| **Jackson 反序列化器** ✅ | 零性能开销、天然支持嵌套对象、字段级精确跳过 | 不覆盖 @RequestParam |
| Filter + HttpServletRequestWrapper | 覆盖所有输入 | 需完整读→解析→序列化 body，性能差；复杂 JSON 易漏 |

**选择 Jackson 方案**：项目 95%+ 的 String 输入走 `@RequestBody`，且不与现有 Sa-Token 拦截器、Jackson 配置产生冲突。

### 清洗规则

- **移除**：`<script>`、`onerror`、`onclick`、`<iframe>`、`<object>`、`<embed>` 等危险标签和事件属性
- **保留**：`<b>`、`<i>`、`<u>`、`<p>`、`<br>`、`<a href>`、`<img src>` 等安全富文本标签（`Safelist.relaxed()`）
- **跳过字段**：`password`、`account`（标注 `@SkipXssClean`，避免修改认证凭据）

### 测试验证

| 输入 | 输出 | 结论 |
| --- | --- | --- |
| `<script>alert(1)</script><p>文本</p>` | `<p>文本</p>` | 危险标签清除，安全标签保留 |
| `<b>加粗</b><a href="...">链接</a>` | 原样保留 | 安全标签不误杀 |
| `<img src=x onerror=alert(1)>` | `<img>` | 事件属性清除 |
| `<script>hack</script>pass123`（password） | 原样保留 | @SkipXssClean 生效 |

### 与后续管道的关系

```
Jackson XSS 清洗 → @Valid 参数校验 → SensitiveWordUtil 敏感词过滤 → Service → DB
```

## 四、敏感词过滤

**目标**：在内容入库前自动检测和替换敏感词，净化社区内容。

### 变更清单

| 文件 | 动作 | 说明 |
| --- | --- | --- |
| `common/util/SensitiveWordUtil.java` | **新建** | DFA（Trie 树）算法 + 内置 25 个敏感词 + FilterResult |
| `post/service/impl/PostServiceImpl.java` | 修改 | title + content 入库前过滤，命中打 warn 日志 |
| `interact/service/impl/CommentServiceImpl.java` | 修改 | content 入库前过滤，命中打 warn 日志 |
| `message/service/impl/MessageServiceImpl.java` | 修改 | content 入库前过滤，命中打 warn 日志 |

### 核心设计

```java
SensitiveWordUtil.filter(text) → FilterResult {
    filteredText:  "这是一条***广告***信息",
    hasSensitive:  true,
    matched:       ["赌博", "博彩"]
}
```

### 技术细节

- **算法**：DFA 有限状态自动机（Trie 树），时间复杂度 O(n)，与词库大小无关
- **匹配策略**：最长匹配原则（`博彩` 优先于 `博`）
- **替换方式**：统一替换为 `***`（固定长度，不暴露原词长度），不做直接删除（避免语句不通）
- **日志记录**：命中时打印 WARN 级别日志，记录 userId 和命中的词列表，便于运维审计

### 内置词库（25 个，分 6 类）

| 类别 | 示例 |
| --- | --- |
| 色情低俗 | 色情、淫秽、裸聊、成人视频、约炮、黄片、av女优 |
| 赌博 | 赌博、博彩、赌场、六合彩、押注、时时彩 |
| 广告引流 | 兼职日结、加微信、扫码领、点击领取、免费红包 |
| 政治敏感 | 法轮功、台独、藏独、疆独、falungong |
| 辱骂攻击 | 傻逼、他妈的、操你妈、去死吧 |
| 违法 | 代办证件、枪支弹药、毒品 |

> 后续可从数据库加载词库，调用 `SensitiveWordUtil.addWords()` 动态扩展。

### 测试验证

| Service | 原始输入 | 入库内容 | 日志 |
| --- | --- | --- | --- |
| PostServiceImpl | title: `这是一条赌博广告博彩信息` | `这是一条***广告***信息` | matched=[赌博, 博彩] |
| PostServiceImpl | content: `加微信免费红包...` | `************快来` | matched=[加微信, 免费红包, 点击领取, 兼职日结] |
| CommentServiceImpl | content: `傻逼去死吧操你妈` | `*********` | matched=[傻逼, 去死吧, 操你妈] |
| MessageServiceImpl | content: `代办证件枪支弹药毒品出售` | 已过滤入库 | matched=[代办证件, 枪支弹药, 毒品] |

## 五、完整请求处理管道

```
HTTP 请求
  │
  ├─ 1. CorsFilter          (跨域放行)
  ├─ 2. SaTokenInterceptor  (JWT 鉴权，白名单放行)
  ├─ 3. @RateLimit AOP      (Redis ZSET 滑动窗口限流)
  │
  ▼
  ├─ 4. Jackson 反序列化
  │     └─ XssStringDeserializer → Jsoup.clean(Safelist.relaxed())
  │     └─ Long → String (前端序列化)
  ├─ 5. @Valid / @Validated (JSR-303 参数校验)
  │
  ▼
  ├─ 6. Controller → Service
  │     └─ SensitiveWordUtil.filter() → DFA 敏感词替换
  │     └─ BusinessException → GlobalExceptionHandler → Result JSON
  │
  ▼
  └─ 7. MyBatis-Plus → MySQL
        └─ MybatisPlusMetaObjectHandler (createdAt/updatedAt 自动填充)
```
