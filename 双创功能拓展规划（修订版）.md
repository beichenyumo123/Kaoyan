# 考研论坛 — 功能拓展规划（修订版）

> 本文档综合了《双创.md》原始条目、修正意见和新增建议，形成完整的功能拓展规划。
> 编号1~14为原始条目（已做修正和细化），15~25为新增建议。

---

## 1. 第三方平台登录
支持微信、QQ 等主流第三方平台 OAuth 2.0 登录，降低注册门槛，提升用户体验。

## 2. 登录安全增强（图形验证码）
- 登录失败 N 次后触发滑块验证码（如极验/腾讯滑块），防止撞库和暴力破解
- 注册时始终需要图形验证码，防止机器批量注册
- 可选接入手机短信验证码二次验证

## 3. 学习进度统计
- 按日/周/月维度展示用户学习时长统计
- 结合现有打卡数据生成可视化图表
- 支持按科目分类统计（如数学、英语、政治、专业课）

## 4. 内容推荐算法
- 基于用户目标专业（已有 `target_major` 字段）推荐相关板块帖子
- 基于用户浏览/点赞/收藏历史做协同过滤
- 热度衰减算法（如 Hacker News 排名算法），避免旧帖长期霸榜
- 新帖冷启动策略，给予一定曝光权重

## 5. 版主管理
当前系统已有 MODERATOR 角色，需明确版主额外权限：
- 管理所辖板块内帖子（置顶、加精、删除、移动）
- 管理所辖板块内违规评论（删除、警告）
- 版主操作日志（可追溯），管理员可查看

## 6. 管理员审核
当前举报系统已初步实现审核流程，需增加：
- 帖子发布审核（先审后发/先发后审两种模式可切换）
- 敏感词自动过滤（集成文本审核服务，如阿里云内容安全）
- 审核结果自动通知用户（站内通知）

## 7. 富文本编辑与渲染（Markdown）
- 帖子编辑器和评论区支持 Markdown 语法（加粗、标题、代码块、数学公式 LaTeX）
- 支持粘贴/拖拽上传图片
- 渲染时需做 XSS 过滤，防止脚本注入

## 8. 小组群聊
- 用户可创建/加入学习小组（如"408 刷题组"、"英语每日打卡组"）
- 组内支持群聊消息、文件分享
- 组长可管理成员、设置小组公告
- 技术选型：可考虑 WebSocket 实时通信

## 9. 学习资料上传
- 支持上传文件（PDF、Word、图片、视频等）
- 按板块分类，支持搜索和下载
- 上传者可为资料设置积分门槛（消耗积分下载）
- 文件存储：小文件本地存储，大文件可对接 OSS（阿里云/腾讯云）
- `resource_file` 表已存在，可直接复用

## 10. 安全防护与数据保护
- **传输层**：全站 HTTPS，API 接口防重放攻击（nonce + timestamp）
- **存储层**：密码 BCrypt 加密（已实现），手机号/邮箱等个人信息 AES-256 加密存储
- **应用层**：防 SQL 注入（MyBatis 参数化查询已有）、防 XSS（富文本内容清洗）、防 CSRF
- **接口层**：限流（Rate Limiting），防止接口被恶意刷量
- **合规**：用户数据导出/删除功能，满足《个人信息保护法》要求

## 11. 高并发优化
聚焦考研场景的具体瓶颈：
- 考研初试成绩公布日/国家线公布日的流量洪峰应对
- 热门帖子评论的高并发写入 → 评论异步落库 + 消息队列
- 帖子列表/排行榜缓存（Redis），避免高频查库
- 数据库读写分离，读操作分担到只读副本

## 12. 题库刷题
- 按科目分类建立题库（数学、英语、政治、408 等专业课）
- 支持单选题、多选题、判断题、填空题
- 用户可随机刷题、按章节刷题、按难度刷题
- 记录做题历史和正确率
- 错题自动归入错题集

## 13. AI 能力集成

### 13.1 AI 对话
- 接入大语言模型（如通义千问、DeepSeek），提供考研领域知识问答
- 针对考研场景做 Prompt 优化，限定回答范围（学习辅导）
- 可选：用户可分享 AI 对话内容到帖子

### 13.2 图片搜题
- 用户上传题目截图 → OCR 识别文字 → 题库检索匹配
- 匹配结果展示题目解析，支持一键收藏到错题集/收藏夹
- **前置依赖**：需先建立题库和收藏夹体系

### 13.3 AI 内容过滤
- AI 自动阅读帖子列表，根据用户关注的专业/板块过滤低相关信息
- 生成每日学习要点摘要推送
- 可根据用户点赞/收藏行为持续优化过滤策略

## 14. 玩家对战
- **做题比赛**：用户匹配或邀请好友进行实时答题对战
- **判题方式**：客观题自动判分，主观题可依赖 AI 评价或双方互评
- **积分制**：胜负结果与现有积分系统联动，计入排行榜
- 可设定比赛科目、题数、时间限制
- 提供对战历史记录和胜率统计

---

## 15. 消息通知系统
当前系统只有私信，缺乏系统通知机制：
- 被回复/被点赞/被收藏/被举报处理结果 → 自动生成通知
- 站内通知列表 + 未读红点角标
- 可选接入邮件通知或微信模板消息推送
- 用户可自定义通知偏好（哪些类型不通知）

## 16. 考研院校信息库
- 院校基本信息：所在地、985/211/双一流标签、官网链接、Logo
- 硕士专业目录：按学科门类、一级学科、二级学科筛选
- 历年招生人数、报录比、推免比例、复试分数线（校线/院线）
- 支持多院校数据对比（表格并排展示）
- 用户可收藏目标院校

## 17. 分数线查询与走势图
- 国家线历年走势（按学科门类，折线图展示）
- 34 所自划线院校复试线逐年对比
- 用户输入目标分数 → 自动匹配可报考院校区间
- 数据来源：教育部官方公布数据 / 人工维护

## 18. 精华帖与经验贴
- 版主或管理员可将优质帖子标记为"精华"，板块内优先展示
- 独立"经验贴"分类，汇集上岸学长学姐备考心得
- 支持按标签筛选（如"二战"、"跨考"、"在职考研"、"双非逆袭"）
- 精华帖可计入用户积分奖励（鼓励高质量内容产出）

## 19. 复习计划与进度追踪
在第3条基础上扩展：
- 用户可创建复习计划（设定科目、目标分数、每日学习任务）
- 按周/月展示学习时长统计图表（结合现有打卡数据）
- 番茄钟计时器（前端功能，专注时长同步到打卡系统）
- 计划完成度可视化（进度条/甘特图）
- 支持"公开计划"，与其他用户互相监督

## 20. 模拟考试
- 管理员/版主可从题库选卷创建模拟考试（设定科目、时长、总分）
- 用户在线作答，倒计时 + 自动交卷
- 客观题自动判分，主观题支持互评或 AI 评阅
- 考试结束后查看排名、每题正确率、错题解析
- 支持"全真模拟"模式（限时、不可暂停、不可切换页面）

## 21. 研友匹配
- 基于目标院校、目标专业、复习阶段等信息匹配相似用户
- 用户可选择公开/隐藏匹配信息（隐私控制）
- 匹配后可通过私信联系或组建学习小组
- 匹配算法可考虑学习活跃度、目标相似度等维度

## 22. 数据看板（后台）
管理员后台分析面板：
- 日活/月活、新增用户趋势、留存率
- 帖子/评论/点赞数量趋势
- 举报处理率、平均审核时长
- 热门板块 TOP 10、热门帖子 TOP 20
- 前端渲染可选用 ECharts / AntV 图表库

## 23. 内容审核自动化
- 可维护敏感词库，支持自定义添加/删除
- 图片内容审核（对接第三方服务检测涉政、色情、暴力内容）
- 用户信用分机制：违规行为扣分，低于阈值限制发帖/评论权限
- 自动审核日志，管理员可追溯每条内容的审核决策

## 24. 移动端适配
- 考研用户以手机为主要使用设备，移动端体验至关重要
- 方案一：Web 前端采用响应式设计（成本低，适配一次即可）
- 方案二：开发微信小程序（生态成熟，可与第1条微信登录联动，分享裂变效果好）
- 方案三：H5 + PWA（渐进式 Web 应用，可添加到手机桌面）
- 推荐优先做小程序，后续考虑响应式 Web 作为补充

## 25. 性能与运维
- 接入 APM 监控（如 SkyWalking、Prometheus + Grafana）
- 接口响应时间监控和慢查询告警
- 日志集中管理（ELK 或阿里云 SLS）
- 定时任务管理（如 XXL-JOB），替代 @Scheduled 注解
- 容器化部署（Docker + K8s），方便弹性扩缩容

# 第二部分：双创视角补充建议

> 以上 25 条偏「功能补全」，对评委而言多数属于「标准论坛功能 + AI 包装」，单看竞争力有限。
> 本部分聚焦双创赛事的核心评价维度——**创新性 × 技术含量 × 商业 / 社会价值 × 可行性**，
> 重新审视已有规划，并提出能让项目形成差异化壁垒的「硬核创新点」与落地路径建议。

## 一、对已有 25 条规划的诊断

**已具创新潜力（建议重点包装、加深技术含量）：**
- 第 13 条 AI 能力集成 → 但当前描述偏「chatbot 套壳」，需升级为「多智能体 Agent」叙事
- 第 14 条 玩家对战 → 游戏化是稀缺亮点，可结合知识图谱做"按知识点匹配对手"
- 第 21 条 研友匹配 → 社交 × 算法的天然结合点

**属于行业基线（建议作为「完善」而非「创新点」展示）：**
- 1、2、7、8、10、11、22、23、24、25 → 任何 SaaS 都该有的工程能力，单独讲不会加分

**有特色但需重写故事的：**
- 16、17 → 院校信息库本身不新，与下文「AI 智能择校引擎」结合才有价值
- 18 → 经验贴若叠加下文「上岸认证体系」可成护城河，否则只是分类标签

## 二、核心创新点建议（按推荐度排序）

### A. AI 多智能体学习伴侣（Multi-Agent Study Companion） ⭐核心
不再是 chatbot，而是**主动型 Agent 团队**：

- **规划 Agent**：根据用户目标院校 / 当前进度 / 剩余时间，动态生成每日任务清单
- **答疑 Agent**：基于考研知识图谱 + RAG，定位问题对应的考点链路
- **监督 Agent**：连续打卡断签或学习时长下滑时主动介入，发送 push
- **心理 Agent**：识别打卡日记的负面情绪，给出安抚 / 引导
- **复盘 Agent**：每周自动生成「本周学情报告」，定位薄弱点

**技术深度**：LangChain4j / Spring AI、Agent 间消息总线、长期记忆（向量数据库）、Tool Calling
**双创亮点**：从「我问 AI 答」升级到「AI 主动陪学」，呼应"教育公平 / 千人千面"叙事

### B. 考研知识图谱 + RAG 智能诊断 ⭐核心
- 把考研大纲 → 知识点 → 真题 → 解析 → 错题 构建成图数据库（Neo4j）
- 用户答错一道题 → 定位到对应知识点 → 沿**前置依赖**反向诊断真实薄弱点
- 推荐学习路径：「你应先掌握 X 再回头做这类题」
- 与第 12 条题库、第 13.2 条图片搜题强联动

**技术深度**：图数据库、知识抽取（DeepKE）、RAG、向量相似度
**双创亮点**：解决「刷了很多题但不知道自己弱在哪」的真实痛点，比单纯「题库刷题」高一个维度

### C. AI 模拟复试官（多模态评估） ⭐核心
复试是考研**最焦虑、信息最不对称**的环节，目前市面上几乎没有产品覆盖：

- 文字模式：模拟英语口语自我介绍、专业课追问、综合面「压力面」
- 语音模式：实时语音对话 + 流利度 / 发音评分
- 视频模式（进阶）：捕捉表情 / 坐姿 / 眼神，给出仪态建议
- 可指定院校风格：清华 vs 同济 vs 普通 985 综合面，问题倾向显著不同
- 输出多维度报告：内容深度 / 语言表达 / 心理状态 / 改进建议

**技术深度**：LLM 角色扮演、ASR（FunASR）、TTS（CosyVoice）、可选 OpenCV 表情识别
**双创亮点**：刚需、稀缺、可独立商业化，是项目最值得讲的核心壁垒之一

### D. 上岸认证 + 可信经验数据库 ⭐核心
- 上岸用户上传**录取通知书 + 学信网截图**进行 OCR / 人工审核认证
- 认证用户的经验贴打「✅ 已认证」金标，作为可信信号
- 经验贴**结构化**字段：本科背景 / 跨考否 / 二战否 / 初试分 / 复试分 / 各科分数 / 用书 / 时间线
- 用户按「我的本科 + 目标院校 + 目标分数」精准检索相似上岸路径

**双创亮点**：信任体系 + 数据沉淀 = 平台护城河
**商业价值**：认证用户可开通「付费 1V1 咨询」（见 H 项），形成 GMV 闭环

### E. 云自习室 / 实时陪伴学习
- WebRTC 多人视频自习室（对标 Cosmos / 番茄 Todo / Forest）
- 摄像头开启即「打卡」，番茄钟同步到第 19 条复习计划与积分系统
- 房间分类：408 区 / 英语区 / 政治区 / 复试备战区
- **房间内禁止聊天**（避免变社交工具），但可发「⏰ 加油」贴纸

**技术深度**：WebRTC（SRS / mediasoup）、Redis Pub/Sub、活体检测防作弊
**双创亮点**：陪伴感 + 仪式感 + 反孤独，命中 Gen-Z 用户的情绪刚需

### F. 抗内卷的「对比组」算法
当前排行榜（第 8 条积分排行）是「全员卷」，会让弱势用户越看越焦虑。建议改造：

- 系统按本科层次 / 目标院校 / 起步阶段，给每位用户匹配 ~50 人的「对比组」
- 用户看到的榜单**只展示对比组数据**，而非全平台
- 增设「进步榜」：本周**进步幅度**最大的用户上榜，而非绝对值最高
- 可双轨开放（全榜 / 对比组），由用户自选

**双创亮点**：心理学 × 算法的产品哲学，对应「反内卷」「教育公平」社会叙事——评委友好

### G. AI 智能择校引擎
依托 D 项积累的**真实上岸数据**：

- 输入：本科院校 / GPA / 英语等级 / 备考时长 / 模考分数 / 风险偏好
- 输出：保底 / 合适 / 冲刺 三档院校推荐 + 「与你相似的 N 位上岸者去了哪些学校」
- 与市面「院校信息库」（第 16、17 条）的区别：用大数据回答「我能去哪」，而非「有哪些学校」

**双创亮点**：从信息工具升级为**决策工具**，形成「用户越多→数据越准→吸引新用户」的数据飞轮

### H. 学长学姐 1V1 付费咨询市场（商业闭环）
- 仅 D 项已认证用户可开通服务
- 服务类型：方向选择 / 用书推荐 / 复试辅导 / 调剂咨询
- 形式：文字咨询包 · 视频通话（30/60 min）· 全程陪跑包月
- 平台抽成 10–20%，可持续商业化
- 评价体系 + 退款保护 + 投诉仲裁

**双创亮点**：把社区流量直接变现为知识付费 GMV，是少有「能上规模赚钱」的考研产品方向；路演时可口算 LTV / GMV / 抽成预期

### I. 学习心理健康守护
- NLP 分析打卡日记 / 私信 / 动态文本，识别焦虑、抑郁、自我否定信号
- 触发后由 A 项的「心理 Agent」介入；严重情况引导到学校心理咨询中心 / 公益援助热线
- 隐私优先：本地化推理 或 脱敏后再调用云端 API
- 提供「焦虑指数曲线」可视化，让用户自我觉察

**双创亮点**：考生心理危机每年都有社会新闻，**社会价值高、政策面友好**，评委极易共情

### J. OCR 智能错题本 + 艾宾浩斯复习计划
在第 13.2 条基础上深化：

- 拍照 → OCR → 知识点定位（B 项知识图谱）→ 入错题本
- 按**艾宾浩斯遗忘曲线**自动推送「今日该复习的错题」
- 错题本支持导出 PDF（便于打印）/ 一键分享给研友
- 错题数据回流到 B 项知识图谱，作为该用户的薄弱点画像

**双创亮点**：把「被动刷题」升级为「主动复盘」，与 B、G 项形成数据闭环

### K. 备考全周期日历与政策智能推送
- 内置考研全周期日历：预报名 / 正式报名 / 现场确认 / 准考证 / 初试 / 出分 / 国家线 / 复试 / 调剂
- 抓取教育部、目标院校官网公告，**针对用户已收藏院校**精准推送
- 与第 15 条通知系统打通

**双创亮点**：相比通用日历，「按目标院校精准过滤」是关键差异点，且数据可作为 G 项择校引擎的输入

## 三、商业模式建议（路演必备「赚钱故事」）

| 收入来源 | 说明 | 上线优先级 |
| --- | --- | --- |
| 付费 1V1 咨询抽成（H 项） | 平台抽成 10–20%，对标 fenbi / 鲸打卡 | 高 |
| 付费精品课 / 资料 | 认证学长学姐上传讲义 / 视频，按下载或会员收费 | 中 |
| 会员订阅（Pro 版） | 解锁 AI Agent 高频调用、无限次模拟复试、错题本 PDF 导出、广告屏蔽 | 中 |
| 院校 / 机构广告位 | B 端：高校研究生院、考研机构定向投放 | 低（流量起量后） |

## 四、实施优先级建议（团队资源有限时的取舍）

**MVP 阶段（双创答辩前必做）：B + C + D 三件套**
- 知识图谱诊断 / 模拟复试 / 上岸认证 — 这三个**最稀缺、最区别于市面产品**
- 路演 demo 集中演示这三个，其他功能（私信、点赞、打卡）一笔带过

**第二阶段：A + E + I**
- 强化「AI + 情感陪伴」产品定位

**第三阶段：G + H**
- 数据飞轮 + 商业闭环成型

**踩坑提醒：**
- 不要把宝押在第 1、2、7、10、24 条（基线能力，评委不加分）
- 不要堆砌过多功能导致故事讲不完整 — 选 3–5 个核心创新点深挖更打动评委
- 慎用「区块链上链经验贴」「元宇宙自习室」等噱头，评委一问就穿帮
- 不必追求「先有题库才有刷题」，可逆向：先做小规模 408 真题 + 知识图谱，跑通 B 项 demo

## 六、双创路演核心叙事建议

1. **痛点切入**：「全国 470 万考研大军，信息不对称、心理焦虑、择校盲目、复试黑箱。」
2. **解决方案重定位**：不是「又一个考研论坛」，而是「**AI 驱动的考研全周期陪伴平台**」
3. **核心创新（讲透 3 个）**：B 知识图谱诊断 · C AI 模拟复试 · D 上岸认证体系
4. **商业模式**：1V1 咨询抽成 + 会员订阅，有可验证收入路径（H 项）
5. **社会价值**：教育公平（信息平权）+ 心理健康（焦虑干预）+ 反内卷文化
6. **数据飞轮**：用户越多 → 上岸数据越多 → 诊断 / 择校越准 → 吸引新用户 → 闭环

---
